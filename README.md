# GoidaCraft Elytra Control

Серверный мод для **Minecraft 1.21.1 / NeoForge 21.1.228+**, меняющий поведение элитр по измерениям.
Работает на выделенном сервере, **не требует клиентских модов** (ванильные клиенты подключаются),
вся логика — на серверном тике, поэтому корректна даже при низком TPS.

## Поведение по измерениям

| Измерение | Поведение |
|---|---|
| **Край** (`minecraft:the_end`) | Элитры полностью запрещены. Нельзя надеть никаким способом (ПКМ, shift-клик, перетаскивание, раздатчик, `/item`, другие моды) — сервер принудительно снимает и возвращает в инвентарь. Любой полёт мгновенно останавливается. |
| **Обычный мир** (`minecraft:overworld`) | Надевать можно, но только **планирование вниз**: вертикальная скорость зажата (всегда чуть вниз), горизонтальная ограничена (блокирует разгон фейерверками и иными способами). Фейерверки в полёте запрещены. |
| **Ад** (`minecraft:the_nether`) | Без ограничений. |

## Сообщения игроку (action bar)

- Край, попытка надеть/лететь → `§c[Элитры] В Крае надевать элитры запрещено!`
- Край, ПКМ элитрой из хотбара → `§c[Элитры] В Крае элитры не работают!`
- Овер, блокировка разгона → `§e[Элитры] Ускорение заблокировано!`
- Овер, начало планирования (одноразово) → `§7[Элитры] Только планирование вниз`
- Овер, фейерверк → `§c[Элитры] Использование фейерверков на элитрах запрещено!`

Сообщения о блокировке не спамят (кулдаун ~2 c); ПКМ-элитра — без кулдауна (явное действие игрока).

## Конфиг

`config/goidacraft_elytra-server.toml` (server-config, синхронизируется с сервера):

| Ключ | По умолчанию | Описание |
|---|---|---|
| `elytraIds` | `minecraft:elytra`, `betterend:elytra_armored`, `betterend:elytra_crystalite` | Список registry-id ограничиваемых элитр (точное совпадение). |
| `autoDetectByName` | `false` | Если `true` — любой предмет с `elytra` в id считается элитрой, **но только** если реально даёт полёт (`canElytraFly`); посторонние предметы игнорируются. |
| `maxHorizontal` | `1.0` | Макс. модуль горизонтальной скорости (блоков/тик). |
| `maxVerticalUp` | `-0.05` | Потолок vy (`vy = min(vy, значение)`); `<= 0` гарантирует снижение. |
| `notifyCooldownTicks` | `40` | Кулдаун повторных сообщений (тики, 20 = 1 c). |

## Сборка

Требуется JDK 21.

```bash
./gradlew build        # Linux/macOS
.\gradlew.bat build    # Windows
```

Готовый мод — `build/libs/goidacraft_elytra-<версия>.jar`. Положить в `mods/` сервера NeoForge 1.21.1.
Для betterend-элитр дополнительно установить мод **Better End** (необязательная зависимость).

## Реализация

Логика разнесена на три слоя в `ElytraEventHandler`:
1. **Превентивный ПКМ** (`PlayerInteractEvent.RightClickItem`) — отмена надевания элитры в Крае и запуска фейерверка в полёте.
2. **Смена экипировки** (`LivingEquipmentChangeEvent`) — ловит любой способ надевания в Крае.
3. **Тик-страховка** (`PlayerTickEvent.Post`) — кламп скорости в овере, остановка полёта и снятие элитры в Крае.

Ключевой момент надёжности: после правки скорости ставится `player.hurtMarked = true`, что форсит
отправку `ClientboundSetEntityMotionPacket` клиенту (механизм ванильного knockback) — клиент обязан
применить серверную скорость. Именно этого не хватало прежнему KubeJS-скрипту.
